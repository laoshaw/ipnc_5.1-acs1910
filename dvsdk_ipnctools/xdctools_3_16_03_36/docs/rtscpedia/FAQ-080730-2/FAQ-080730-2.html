FAQ-080730-2

How can I change the version of XDCtools used to control the Eclipse plug-ins?

Code Composer Studio 4 references XDCtools and DSP/BIOS versions in different places for different purposes. The two purposes are:

  • IDE support version. The "Eclipse plug-ins" come from the version selected for IDE support. This includes the version for the Runtime Object Viewer and the dialogs that allow you to control RTSC aspects of the build. For details, see below.
  • RTSC Target Runtime support version. Your application is build with this version of XDCtools. This includes the xdc.runtime package, other packages, and tools such as configuro. For details, see FAQ-080729-1 (existing projects) and FAQ-080729-2 (defaults for new projects).

This distinction is useful because you may want to use the latest dialogs and tools within CCStudio for creating and debugging the application, while using an older version to build your application for stability or backward compatibility reasons.

Steps

To set the version of XDCtools that provides the plug-ins necessary for Eclipse integration support, follow these steps:

  1. Choose Help->Software Updates->Manage Configuration from the CCStudio menus.
  2. If the software version you want to use is not listed in the Product Configuration dialog, click Add an Extension Location.
  3. In the "Browse for Folder" dialog, browse to the eclipse subdirectory of the installation directory for XDCtools, and click OK. If you are asked to restart the workbench, please click No at this time.
  4. In the left-hand pane, select the RTSC / XDCtools (IDE Support) product version that you used until now. In the right-hand pane, click "Disable". A dialog will ask whether you'd like to disable the feature, please click OK.
  5. You may receive a warning that The current configuration contains errors and this operation can have unpredictable results. Do you want to continue?. Please click OK.
  6. A dialog will appear that asks you to restart the workbench. Please click Yes.

Unless instructed otherwise, you should always select the latest version of XDCtools for the Eclipse plug-ins.

Upgrading the XDCtools IDE Support version will not upgrade the version of XDCtools used for Target Runtime Support in any existing CCSv4 projects. You must change the version in the project if you want to use the new version for target content, see FAQ-080729-1.
